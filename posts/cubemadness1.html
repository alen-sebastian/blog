HTB CubeMadness1 — Cheating My Way to the Flag

A GamePwn challenge where the goal is to collect 20 cubes in a game that only has 6. Spoiler: you're not meant to find the rest.

I'll be honest — when I first saw the GamePwn category on HTB I almost skipped it. I do web stuff mostly. Game hacking felt like a completely different world. But CubeMadness1 is rated Very Easy and I had twenty minutes to kill, so I downloaded the zip and opened it up. What followed was actually one of the more satisfying "oh it's that simple?" moments I've had in a while.

What even is this challenge

You extract the zip and get a folder full of DLLs, a couple of executables, and some Unity game assets. Running the main executable opens up a small 3D game — you're a little cube-character on a platform, and there are other cubes scattered around the map. The score counter in the top-left reads 0 / 20.

I ran around the map collecting everything I could find. Got to 6. Map was empty. 6 / 20 sitting there mocking me. There are literally no more cubes anywhere on the map.

So the game is lying to you — or rather, it's telling you the win condition is 20 but only giving you the tools to reach 6. Which means you're not supposed to find the other 14. You're supposed to make the game think you already have them.

My first instinct (the wrong one)

My first thought was to reverse the game binary. The game's built with Unity, and when I poked at GameAssembly.dll, I could tell pretty quickly it was compiled with il2cpp. That's Unity's way of converting C# game logic to native C++ — developers use it specifically to make the game harder to reverse. Tools like Il2CppDumper can help, but for a Very Easy challenge this felt like overkill.

I closed dnSpy and opened Cheat Engine instead.

Attaching to the process

With the game running, I opened Cheat Engine and hit the process selector — the flashing monitor icon at the top left. A list of running processes comes up. I selected the CubeMadness1 process and hit Open.

Initial scan — fishing in 40 million addresses

My current cube count is 0. I set the scan type to Exact Value, value type to 4 Bytes (standard integer — most game scores are stored this way), typed 0, and hit First Scan.

Obviously everything in the process that's currently zero shows up — about 40 million addresses. This isn't a problem, it's just the baseline. The narrowing starts now.

Narrowing it down — the good part

I went back into the game and collected exactly one cube. Counter: 1 / 20. Back in CE, I changed the scan type to Increased value by..., entered 1, and hit Next Scan. CE threw out every address that didn't increase by exactly 1.

Collected a second cube. Next Scan again. Down to a few thousand. Kept going — third, fourth cube. After collecting all 6 available cubes and filtering at each step, I was left with just 3 addresses.

All three held the same current value (6) and had changed consistently with my cube count through every single scan. That's it — Unity sometimes keeps the same game variable mirrored across a few memory regions, which is why we get more than one address. Doesn't matter, we change all of them.

Changing the value — the "exploit"

I selected all three addresses in the results panel, hit the red arrow to send them to the address list at the bottom, then double-clicked each value field and changed it to 20.

Switched back to the game window. The counter instantly jumped to 20 / 20 and the flag appeared on screen. 

HTB{CU83_M4DN355_UNM4DD3N3D}

Why this works

Games track state in memory like any other program — they just do it fast and in real time. The cube count is just an integer. There's no server validating it, no checksum, no anti-cheat layer. We're running this locally, so we own the process. Cheat Engine just gives us a systematic way to find which integer it is among the millions sitting in memory.

The multi-pass scanning technique (scan for 0 → collect → scan for 1 → collect...) is the core of how Cheat Engine narrows things down. Every time the value changes by a known amount, you filter out everything that didn't match. After a handful of iterations you're left with the exact addresses you care about.

Would I try CubeMadness2?

Yeah, already did. The second challenge in the series adds encryption on top of the memory values — the raw integer in CE isn't the actual cube count, it's an encoded form of it. Same concept, significantly more annoying to pull off. I'll write that one up soon.

This one though — genuinely a good intro to GamePwn. If you've never touched Cheat Engine before, this challenge teaches you the fundamentals of memory scanning in about twenty minutes, which is a solid return on time investment.

If you're new to game hacking, the Cheat Engine built-in tutorial (it ships with one) is actually worth going through before trying CTF challenges. It covers exactly this kind of value scanning. Questions? Find me on Twitter or email.
